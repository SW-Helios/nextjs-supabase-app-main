> **작성일**: 2025-10-13
> **버전**: MVP v1.0
> **기반 문서**: [PRD.md](./PRD.md)

## 📋 린캔버스 개요

린캔버스는 비즈니스 모델을 한 페이지에 간결하게 정리한 전략 도구입니다.
Gather 플랫폼의 핵심 가치와 실행 전략을 9가지 블록으로 구성했습니다.

---

## 1. 문제 (Problem)

**Top 3 해결해야 할 문제**

1. **이벤트 관리의 복잡성**
   - 생일 파티, 워크샵, 세미나 등 소규모 일회성 이벤트를 관리하기 위해 여러 도구를 사용해야 함
   - 초대장, 참여자 명단, 일정 공유 등을 개별적으로 관리하는 것이 번거로움

2. **실시간 참여 현황 파악 어려움**
   - 누가 참여하는지, 몇 명이 오는지 실시간으로 확인하기 어려움
   - 카카오톡, 문자 등으로 개별 확인해야 하는 불편함

3. **참여 장벽이 높음**
   - 참여하려면 복잡한 앱 설치나 회원가입 절차를 거쳐야 함
   - 초대 링크만으로 간편하게 참여할 수 있는 방법 부족

**기존 대안 (Existing Alternatives)**
- 카카오톡 단체 채팅방 (수동 관리, 기능 부족)
- 구글 캘린더 (일정 공유만, 참여자 관리 불가)
- Eventbrite, 소모임 등 (대규모 이벤트용, 복잡함)

## 2. 고객 세그먼트 (Customer Segments)

**타겟 고객**

### 주 고객 (Primary)
- **5-30명 규모 이벤트 주최자**
  - 생일 파티, 동창회, 스터디 모임, 워크샵, 세미나 등을 주최하는 개인
  - 연령대: 20-40대
  - 특징: 모바일 사용에 익숙, 간편한 관리 도구 선호

### 부 고객 (Secondary)
- **이벤트 참여자**
  - 초대 링크를 통해 이벤트에 참여하는 사용자
  - 특징: 빠르고 간단한 참여 프로세스 선호

### 얼리어답터 (Early Adopters)
- 20-30대 직장인 및 대학생
- SNS 활동이 활발하고 새로운 서비스를 빠르게 시도하는 사용자
- 모바일 앱/웹 서비스 사용에 익숙한 사용자

## 3. 가치 제안 (Unique Value Proposition)

**한 줄 요약**

> **"초대 링크 하나로 3초 만에 끝내는 소규모 이벤트 관리"**

**핵심 가치**
- ⚡ **3초 만에 로그인**: Google 계정으로 즉시 시작
- 🔗 **초대 링크 하나로 끝**: 복잡한 절차 없이 링크 하나로 모든 것 해결
- 👥 **실시간 참여자 목록**: 누가 참여하는지 실시간으로 확인
- 📱 **모바일 최적화**: 언제 어디서나 쉽고 빠르게 관리

**고객이 얻는 혜택**
- 이벤트 생성부터 참여자 관리까지 **5분 이내 완료**
- 참여자 확인을 위해 **일일이 연락할 필요 없음**
- 카카오톡, 문자로 **간편하게 초대 링크 공유**

---

## 4. 해결책 (Solution)

**Top 3 핵심 기능**

1. **이벤트 생성 & 초대 링크 자동 생성**
   - 제목, 날짜, 장소, 커버 이미지만 입력하면 초대 링크 자동 생성
   - 생성된 링크를 카카오톡, 문자, 클립보드 복사로 간편 공유

2. **실시간 참여자 목록**
   - Supabase Realtime을 활용한 실시간 참여자 목록 업데이트
   - 참여자 프로필 사진, 이름, 역할 표시

3. **초대 링크 참여 (원클릭 참여)**
   - 초대 링크 클릭 → 이벤트 미리보기 → Google 로그인 → 자동 참여
   - 이미 로그인한 사용자는 즉시 참여 처리

**추가 기능 (Phase 2)**
- 정산 관리 (N분의 1 자동 계산)
- 카풀 관리 (운전자 등록, 탑승자 배정)

---

## 5. 경쟁우위 (Unfair Advantage)

**쉽게 모방할 수 없는 우리의 강점**

1. **일회성 이벤트 특화**
   - 경쟁사(소모임, 카카오 톡캘린더 등)는 정기 모임 또는 일정 공유에 집중
   - Gather는 **일회성 이벤트에 최적화된 UX**

2. **모바일 퍼스트 + 깔끔한 디자인**
   - 럭키밀(luckymeal.io) 스타일의 직관적이고 아름다운 카드 UI
   - 터치 영역 최적화, 하단 내비게이션 바 등 모바일 최적화

3. **실용적 관리 기능 (Phase 2)**
   - 정산 관리 (N분의 1 자동 계산, 송금 체크리스트)
   - 카풀 관리 (운전자/탑승자 매칭)
   - 경쟁사에 없는 **실생활 밀착형 기능**

4. **한국 시장 최적화**
   - 카카오톡 공유 최적화
   - 한국어 UI/UX
   - 완전 무료 제공 (광고 없음)

5. **Next.js 15 + Supabase 최신 기술 스택**
   - 빠른 성능과 확장성
   - Realtime 기능으로 즉각적인 업데이트

## 6. 수익 흐름 (Revenue Stream)

**수익 모델**

### MVP 단계 (Phase 1)
- **완전 무료 제공**
- 목표: 사용자 확보 및 피드백 수집
- 수익화 없음 (성장에 집중)

### Phase 2 이후 (수익화 검토)
1. **프리미엄 구독 모델 (월 구독)**
   - 무료 플랜: 기본 이벤트 생성 및 관리
   - 프리미엄 플랜 (월 4,900원):
     - 정산 관리 고급 기능 (자동 계산, 송금 리마인더)
     - 카풀 관리 고급 기능 (GPS 경로 최적화)
     - 이벤트 참여자 수 제한 해제 (30명 → 100명)
     - 커스텀 초대장 템플릿

2. **기업용 플랜 (B2B)**
   - 워크샵, 세미나, 네트워킹 이벤트 주최하는 기업 대상
   - 월 19,900원 (무제한 이벤트, 관리자 기능, 통계 리포트)

3. **광고 (최후 수단)**
   - 무료 사용자 대상 비침해적 광고 (선택적)

**예상 수익 (Phase 2 출시 후 3개월)**
- MAU 1,000명 기준
- 프리미엄 전환율 10% → 100명 × 4,900원 = 월 49만원

## 7. 비용 구조 (Cost Structure)

**주요 비용 항목**

### 고정 비용
1. **인프라 비용**
   - Supabase (무료 → 월 $25 Pro 플랜)
   - Vercel (무료 → 월 $20 Pro 플랫폼)
   - 도메인 비용 (연 $15)

2. **개발 비용**
   - 1인 개발자 (6주 Sprint, MVP 출시)
   - Phase 2 개발 (3개월)

### 변동 비용
1. **스토리지 비용** (사용자 증가에 따라)
   - 커버 이미지 업로드 (Supabase Storage)
   - 예상: 사용자당 평균 5MB → 1,000명 기준 5GB

2. **데이터베이스 비용** (트래픽 증가에 따라)
   - Supabase 무료 플랜: 500MB 데이터베이스, 1GB 파일 스토리지
   - 유료 전환 시점: MAU 1,000명 돌파 시 ($25/월)

**MVP 단계 총 비용 예상**
- 개발 기간: 6주 (1인 개발)
- 인프라 비용: $0 (Supabase + Vercel 무료 플랜)
- 도메인: $15/년
- **총 비용: ~$15 (도메인 비용만)**

## 9. 채널 (Channels)

**고객 접점 및 유입 경로**

### 주요 채널 (Primary)
1. **카카오톡 초대 링크 공유** ⭐ (가장 중요)
   - 주최자가 초대 링크를 카카오톡으로 공유
   - 참여자가 링크 클릭 → 바이럴 효과

2. **문자 메시지 (SMS)**
   - 카카오톡 미사용자 대상
   - 초대 링크 복사 후 문자로 공유

3. **웹 검색 (SEO)**
   - "소규모 이벤트 관리", "생일 파티 초대" 등 키워드 최적화
   - 블로그 콘텐츠 마케팅

### 보조 채널 (Secondary)
4. **소셜 미디어 (SNS)**
   - 인스타그램, 페이스북 광고 (저비용)
   - 사용자 리뷰 및 성공 사례 공유

5. **커뮤니티 & 포럼**
   - 네이버 카페, 오픈채팅방 등에서 홍보
   - 대학 동아리, 직장인 커뮤니티 타겟

6. **입소문 (Word of Mouth)**
   - 참여자가 주최자가 되는 순환 구조
   - 이벤트 참여 후 "나도 만들어보기" 유도

### 채널 전략
- **바이럴 루프 (Viral Loop)**: 초대 링크 → 참여 → 새로운 주최자 전환
- **네트워크 효과**: 참여자가 많을수록 플랫폼 가치 증가
